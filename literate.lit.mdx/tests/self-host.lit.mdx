# Self-Hosting Check — Bootstrap Proof

**A compiler self-hosts when it can compile itself and produce a bit-identical copy of its own binary.** GCC bootstraps in three stages: Stage 1 (the committed compiler) builds Stage 2 (the compiler compiled by Stage 1), which builds Stage 3 (the compiler compiled by Stage 2). If Stage 2 and Stage 3 are bit-identical, self-hosting is proved.

literate-state-machine-wiki has the same bootstrap problem. The committed files at the repo root — `flake.nix` and `lib/*.nix` — are the Stage 1 bootstrap. They are generated by entangled from `literate.lit.mdx/flake.lit.mdx` and `literate.lit.mdx/lib/*.lit.mdx`, but they must exist on disk at nix-eval time because the flake imports them. This is the same chicken-and-egg as GCC: you need a compiler to build the compiler.

This test proves self-hosting. It tangles the literate source through the library's own pipeline (pulling entangled from the flake input), then compares the result bitwise against the committed bootstrap files. If they match, the library can re-generate itself; if they drift, the committed bootstrap is stale and the library is no longer self-hosting.

## Why bitwise comparison matters

A "close enough" comparison would allow whitespace drift, comment drift, and eventually semantic drift — exactly the kind of erosion that breaks trust in generated code. Bitwise means: every byte of `lib/checks.nix` at the repo root matches every byte of `lib/checks.nix` produced by tangling `literate.lit.mdx/lib/checks.lit.mdx`. If a developer edits the generated file directly (instead of the literate source), this test catches it. If entangled's output format changes between versions, this test catches it. If someone accidentally commits a stale bootstrap, this test catches it.

## What's checked

The bootstrap layer is exactly seven files:

- `flake.nix`
- `lib/checks.nix`
- `lib/pipeline.nix`
- `lib/config.nix`
- `lib/devshell.nix`
- `entangled.toml` (generated from `config.nix`'s `defaultEntangledToml`)
- `README.md` (symlink, not tangled — excluded from byte comparison)

Each non-symlink bootstrap file must match byte-for-byte. The test runs `entangled tangle --force` in a sandboxed copy of the literate source, then diffs each file against the committed version. A single byte of drift fails the check.

## The implementation

We build the literate source through `pipeline.tangle` (which is what `makeVerify` uses internally), producing a tangled tree in the nix store. Then we diff the bootstrap files against the committed versions. `diff -q` is used because we don't need human-readable diffs — pass/fail is enough for CI, and when a developer debugs a failure they can drop to `diff` themselves.

```{.nix file=tests/self-host.nix}
# Generated from literate.lit.mdx/tests/self-host.lit.mdx — DO NOT EDIT
{ pkgs, lib, src, pipelineLib }:

let
  # Tangle the literate source using the library's own pipeline.
  # This is Stage 2 in the bootstrap — the library building itself.
  stage2 = pipelineLib.tangle {
    inherit pkgs src;
    stripGeneratedMarkers = false;  # preserve markers for byte comparison
  };

  # Files that must match bitwise between committed bootstrap and stage2 tangle.
  bootstrapFiles = [
    "flake.nix"
    "lib/checks.nix"
    "lib/pipeline.nix"
    "lib/config.nix"
    "lib/devshell.nix"
  ];

  diffCommands = builtins.concatStringsSep "\n" (map (file: ''
    # stderr visible so "file not found" is distinguishable from "content diff"
    if ! diff -q "${src}/${file}" "${stage2}/${file}" > /dev/null; then
      echo "❌ DRIFT: ${file} differs between committed bootstrap and tangled output"
      echo "   Committed:  ${src}/${file}"
      echo "   Tangled:    ${stage2}/${file}"
      echo ""
      echo "   First 20 lines of diff:"
      diff "${src}/${file}" "${stage2}/${file}" | head -20 || true
      exit 1
    fi
    echo "✅ ${file}"
  '') bootstrapFiles);

in
pkgs.runCommand "literate-state-machine-wiki-self-host-check" { } ''
  echo "=== Self-hosting check: committed bootstrap vs tangled output ==="
  ${diffCommands}

  # README.md must be a symlink to the canonical literate source. If a contributor
  # replaces it with a real file, the bootstrap structure has silently diverged.
  if [ ! -L "${src}/README.md" ]; then
    echo "❌ STRUCTURE: README.md is not a symlink"
    echo "   Expected: symlink to literate.lit.mdx/index.lit.mdx"
    exit 1
  fi
  echo "✅ README.md (symlink)"

  echo ""
  echo "✅ Self-hosting verified: all bootstrap files match bitwise"
  touch "$out"
''
```

## What happens when this fails

If `nix flake check` reports `self-host-check` failing, a developer edited a tangled file directly instead of its literate source. The fix is:

1. Find which file drifted (the error message names it)
2. Open the corresponding `literate.lit.mdx/**/*.lit.mdx` source
3. Make the edit in the literate source
4. Run `entangled tangle --force` to regenerate the committed bootstrap
5. Commit both the `.lit.mdx` and the regenerated `.nix`/`.toml`

The "both files in one commit" rule is what keeps the bootstrap honest. Git history then shows the literate source change alongside the machine-regenerated output — reviewers can verify they match, and future tangles will produce the same output.
