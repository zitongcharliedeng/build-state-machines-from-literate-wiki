# build-state-machines-from-literate-wiki

Tangle `.lit.md` source → immutable nix store output → verify per-language.

One flake input. One command: `nix flake check`. Everything else is derived.

## What it does

1. Tangles `.lit.md` files via [Entangled](https://github.com/entangled/entangled.py) (bundled)
2. Stores output in `/nix/store/` (immutable, content-addressed, cached)
3. Verifies output per-language (tsc for .ts, nix-instantiate for .nix, cargo for .rs)
4. Discovers XState machines in TypeScript output
5. Detects unknown files not produced by any `.lit.md`

## Usage

Add to your `flake.nix`:

```nix
{
  inputs.lsmw.url = "github:zitongcharliedeng/build-state-machines-from-literate-wiki";

  outputs = { self, lsmw, nixpkgs }: let
    pkgs = nixpkgs.legacyPackages.x86_64-linux;
  in {
    # Tangle your source directory → nix store
    packages.default = lsmw.lib.tangle {
      src = ./ONLY_EDIT_THIS_DIRECTORY_EVERYTHING_ELSE_IS_GENERATED_FROM.lit.md;
    };

    # All verification in one command
    checks.x86_64-linux = lsmw.lib.makeChecks {
      src = ./ONLY_EDIT_THIS_DIRECTORY_EVERYTHING_ELSE_IS_GENERATED_FROM.lit.md;
      inherit pkgs;
      languages = {
        typescript = true;  # enables: tsc + ast-grep + XState machine discovery
        nix = true;         # enables: nix-instantiate + module assertions
      };
    };
  };
}
```

Then: `nix flake check`. That's your entire CI.

## Project structure

```
my-project/
├── ONLY_EDIT_THIS_DIRECTORY_EVERYTHING_ELSE_IS_GENERATED_FROM.lit.md/
│   └── ... (your .lit.md files — THE source of truth)
├── flake.nix    ← tangled from .lit.md, committed (bootstrap, drift-checked)
├── flake.lock   ← auto-generated by nix
└── .github/workflows/ci.yml  ← tangled, committed (3 lines)
```

Everything outside the source directory is a derived artifact. Immutable in the nix store. Not hand-edited. Drift-checked.

## Non-Nix fallback

For systems without Nix installed, the TypeScript CLI still works:

```bash
bun run src/cli.ts build .
```

This tangles locally, chmod 444, and reports. Less guarantees than the Nix path but functional.

## Mathematical Foundations

### Coverage guarantee

For XState machines discovered in TypeScript output, `@xstate/graph` provides:

**Shortest path coverage**: For every reachable state S and every enabled event E in state S, there exists a test path that reaches S and fires E. This is computed via Dijkstra's algorithm on the state graph.

**Coverage metrics**:
- **State coverage**: every reachable state is visited by at least one test path
- **Transition coverage**: every (state, event) → state transition is exercised
- **Guard coverage**: every guard condition is evaluated to both `true` and `false` across test paths

This is equivalent to **MC/DC (Modified Condition/Decision Coverage)** for the state machine's guard predicates — the standard required by DO-178C for aviation software.

### Combinatorial trust model

Each dependency in the build chain has an explicit trust level:

```
Trust(system) = ∏ Trust(component_i)
```

| Component | Trust | Source | What could fail |
|-----------|-------|--------|-----------------|
| Entangled tangle correctness | 0.95 | eScience Center, 2.4.2, Apache-2.0 | Tangling bug corrupts output |
| Nix store immutability | 0.999 | NixOS foundation, 20+ years | Store corruption (hardware) |
| Nix derivation hermeticity | 0.99 | Nix sandbox design | Sandbox escape |
| TypeScript compiler (tsc) | 0.999 | Microsoft, millions of users | Type system unsoundness |
| XState runtime | 0.99 | Stately.ai, funded, active | State transition bug |
| @xstate/graph path gen | 0.98 | Dijkstra's algorithm impl | Missing edge case in traversal |

**Combined trust** for a fully-verified TypeScript project:
```
0.95 × 0.999 × 0.99 × 0.999 × 0.99 × 0.98 ≈ 0.91
```

This means: "we are 91% confident the system is correct, given all tools work as trusted." The 9% gap is where integration tests, property-based testing (fast-check), and LLM-as-judge visual testing add value.

### What the flake MUST verify (non-negotiable checks)

| Check | What it proves | How |
|-------|---------------|-----|
| Tangle succeeds | All `.lit.md` files are valid Entangled source | `entangled tangle --force` exits 0 |
| Tangle is idempotent | No hidden state in tangling | Tangle twice, diff outputs |
| No unknown files | Everything outside source dir is accounted for | Compare filedb against filesystem |
| Per-language typecheck | Output is well-typed | tsc --noEmit / nix-instantiate / cargo check |
| Per-language lint | Output follows conventions | ast-grep rules per language |
| Machine discovery | All XState machines found | Grep for createMachine/setup exports |
| Machine coverage | All states/transitions tested | @xstate/graph shortest paths |
| Drift detection | Committed bootstrap files match source | Re-tangle, diff against committed flake.nix/ci.yml |

### What we explicitly DO NOT verify (trusted boundaries)

| Boundary | Why trusted | Trust level |
|----------|-------------|-------------|
| Nix evaluates correctly | Foundational — if Nix is broken, nothing works | 0.999 |
| Hardware stores bits correctly | Below our abstraction layer | 0.9999 |
| LLM wrote correct .lit.md | Human reviews intent; LLM is the editor, not the spec | N/A (human judgment) |
