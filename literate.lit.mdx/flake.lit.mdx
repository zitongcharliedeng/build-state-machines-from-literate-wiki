# Flake

This is the entry point. It imports the modules from `lib/` and wires them together. The flake itself is literate — tangled from this `.lit.mdx` file by Entangled.

The bootstrap order: `entangled tangle --force` produces `flake.nix` from this file. Then `nix flake check` uses that `flake.nix` to tangle, check, and build. The trusted base is Entangled. Everything above it is literate.

The flake is deliberately thin. All logic lives in the modules:
- [[lib/config]] — default entangled.toml, helper functions
- [[lib/pipeline]] — tangling, locking, nix store installation
- [[lib/checks]] — pre-tangle, post-tangle, and structural validation
- [[lib/devshell]] — development environment with auto-tangle

``` {.nix file=flake.nix as-real-file-path="nix requires flake.nix in repo (bootstrap paradox)"}
{
  description = "literate-state-machine-wiki — my opinionated tangle";

  inputs = {
    nixpkgs.url = "nixpkgs";
    entangled.url = "github:entangled/entangled.py";
  };

  outputs = { self, nixpkgs, entangled }:
    let
      system = "x86_64-linux";
      pkgs = nixpkgs.legacyPackages.${system};
      lib = nixpkgs.lib;

      config = import ./lib/config.nix { inherit lib; entangledInput = entangled; };
      pipeline = import ./lib/pipeline.nix { inherit lib config; };
      checksLib = import ./lib/checks.nix { inherit lib config pipeline; };
      devshellLib = import ./lib/devshell.nix { inherit lib config; };

      init = {
        pkgs,
        src,
        system ? "x86_64-linux",
        linters ? [ ],
        sourceDir ? "literate.lit.mdx",
        forbidTsComments ? true,
        minProseLines ? 3,
        maxBlockLength ? 50,
        enforceDirectoryMatch ? false
      }:
      {
        checks.${system} = checksLib.makeChecks {
          inherit pkgs src sourceDir forbidTsComments minProseLines maxBlockLength enforceDirectoryMatch;
          postTangleChecks = linters;
        };

        packages.${system} = {
          tangled = pipeline.tangle { inherit pkgs src; };
          web-wiki = pipeline.buildWebWiki { inherit pkgs src; litSourceDir = sourceDir; };
        };
      };
    in
      # Self-application: this flake uses itself
      (init {
        inherit pkgs;
        src = ./.;
        sourceDir = "literate.lit.mdx";
        maxBlockLength = 200;
      }) // {
        lib = { inherit init; inherit (config) defaultEntangledToml; };

        # DevShell is for literate-state-machine-wiki development ONLY.
        # Consumers do not need this — they use nix flake check.
        # This exists because bootstrapping requires entangled in PATH
        # to tangle flake.nix from its literate source.
        devShells.${system}.default = devshellLib.mkDevShell {
          inherit pkgs;
        };
      };
}
```

One function: `init`. Consumer calls it, gets `checks`, `devShells`, and `packages`.
